What is MCP?
02 / 12

Agent tools bridge

MCP gives Codex named tools instead of asking it to guess where documents live.

Agent asks Natural language becomes a tool call.
Server gates Auth and RBAC happen before content leaves.
Problem
03 / 12

Documentation exists. Finding the right slice is the hard part.

Scattered

PDFs, source notes, runbooks, HTML, and Markdown sit in different places.

Hard to search

People remember config keys, error strings, and commands - not page titles.

Access-sensitive

Search must respect team boundaries before the answer is generated.

Core idea
04 / 12

Turn every source into one curated, searchable store.

The server stays read-only; ingest does the conversion and indexing work.

Source raw docs PDF, Office, HTML, Markdown, code, text
Pipeline ingest Docling, OCR, tree-sitter, text parsers
Output curated Markdown stable logical paths under the doc store
Index keyword search ripgrep or SQLite FTS5; vector optional
Agent MCP tools list, search, read, cite 
# Retrieval path
raw/ -> ingest -> curated/ + index -> list_docs / search_docs / read_doc
Core tools
05 / 12

Small tool surface, predictable results.

list_docs(path="")

Discover

Returns visible docs under a prefix: path, title, type, bytes, mtime.

search_docs(query)

Find exact terms

Keyword/full-text hits: path, line, snippet, score.

read_doc(path)

Open the source

Reads a bounded document or line range; denies disallowed paths.

semantic_search is optional

Vector search only runs when explicitly enabled; default retrieval is keyword-first.

All tools are prefix-filtered

A user token sees only its allowed paths; denied reads fail closed.

Architecture
06 / 12

Caddy fronts the read-only MCP server.

Only Caddy is exposed. The app reads curated docs and indexes; ingest is the writer.

Token + RBAC Bearer token resolves to user, groups, and allowed prefixes.
Search index Keyword by default; vector layer is off unless configured.
Local model
07 / 12

Same machine. Loopback URL.

Codex and the server run on one laptop or workstation.

codex mcp add docs \
  --url http://localhost/mcp \
  --bearer-token-env-var DOCS_MCP_TOKEN
Remote model
08 / 12

Team server over VPN or HTTPS.

Choose the exposure model that matches the trust boundary.

Trusted VPN / LAN

Raw IP is simple for internal networks you control.

http://10.0.0.5/mcp

Public or untrusted network

Use a hostname and HTTPS through Caddy.

https://docs-mcp.company.internal/mcp
Deploy and setup
09 / 12

One helper script for the common path.

Operator
./docmcp.sh setup
./docmcp.sh add /path/to/docs
./docmcp.sh ingest
./docmcp.sh serve
./docmcp.sh token alice /public /team-fw
Codex client
export DOCS_MCP_TOKEN=tok_alice_xxxx

codex mcp add docs \
  --url http://10.0.0.5/mcp \
  --bearer-token-env-var DOCS_MCP_TOKEN

Setup

Build images, create config, mint admin token.

Ingest

Convert sources and build the searchable store.

Serve

Start docs-mcp and Caddy in the background.

Optional capabilities
11 / 12

Grow the workflow when the team needs it.

Portal

Browser upload and manage flow for teammates who do not use git.

Scheduled ingest

Rebuild incrementally on a cron interval; unchanged files are skipped.

Groups and RBAC

Map teams to prefixes once; tokens inherit group access live.

./docmcp.sh schedule 30m
./docmcp.sh group firmware /team-fw
./docmcp.sh access-check alice /team-fw/flashing.md
